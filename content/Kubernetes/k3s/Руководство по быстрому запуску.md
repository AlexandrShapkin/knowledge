## [Скрипт установки](https://docs.k3s.io/quick-start)
K3s предоставляет скрипт установки, удобный для инсталляции как сервиса в системах на базе systemd или openrc. Скрипт доступен по адресу: [https://get.k3s.io](https://get.k3s.io)
```bash
curl -sfL https://get.k3s.io | sh -
```
После запуска установки:
- Сервис K3s будет настроен на автоматический перезапуск после перезагрузки узла или при сбое/завершении процесса.
- Будут установлены дополнительные утилиты: `kubectl`, `crictl`, `ctr`, `k3s-killall.sh` и `k3s-uninstall.sh`.
- Файл `kubeconfig` будет записан в `/etc/rancher/k3s/k3s.yaml`, а установленный вместе с K3s `kubectl` будет использовать его автоматически.

Установка на один серверный узел создаёт полностью функциональный кластер Kubernetes, включая все необходимые компоненты: хранилище данных, control plane, kubelet и контейнерный runtime для запуска рабочих pod-ов. Добавлять дополнительные серверные или агентские узлы необязательно, но это можно сделать для увеличения ресурсов или отказоустойчивости кластера.

Чтобы подключить дополнительные агентские узлы к кластеру, нужно запустить установочный скрипт с переменными окружения `K3S_URL` и `K3S_TOKEN`. Пример:
```bash
curl -sfL https://get.k3s.io | K3S_URL=https://myserver:6443 K3S_TOKEN=mynodetoken sh -
```
Указание параметра `K3S_URL` заставляет установщик настроить K3s как агент, а не сервер. Агент K3s зарегистрируется на сервере K3s, слушающем по указанному адресу. Значение для `K3S_TOKEN` хранится в файле `/var/lib/rancher/k3s/server/node-token` на вашем серверном узле.
> **Примечание:**  
> Каждая машина должна иметь уникальное имя хоста. Если имена хостов не уникальны, укажите переменную окружения `K3S_NODE_NAME` и задайте для каждого узла корректное и уникальное имя.
