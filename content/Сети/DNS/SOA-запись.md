SOA-[[Виды DNS-записей|запись]] содержит базовую информацию о зоне DNS и задаёт «начало авторитета». Она обязательна для каждой зоны.

**Формат (zone-файл):**
```
example.com. 86400 IN SOA ns1.example.com. admin.example.com. (
    2025093001 ; serial
    3600       ; refresh
    900        ; retry
    1209600    ; expire
    86400 )    ; minimum
```

## Поля SOA:
1. **Primary NS** — главный авторитетный сервер зоны.
2. **Responsible party** — e-mail администратора (знак `@` заменяется на точку).
3. **Serial** — номер версии зоны; увеличивается при изменениях.
4. **Refresh** — как часто вторичные NS должны запрашивать обновления.
5. **Retry** — как часто вторичные NS пробуют повторно, если основной недоступен.
6. **Expire** — время, после которого вторичные NS перестают обслуживать зону без обновления.
7. **Minimum (TTL)** — минимальное время кеширования для записей без явного TTL (в старых реализациях — TTL для NXDOMAIN).

## Сценарии использования:
- Контроль синхронизации между primary и secondary DNS-серверами.
- Управление кешированием и консистентностью зоны.

## Особенности и ограничения:
- Всегда одна SOA-запись на зону.
- При изменении любых записей в зоне нужно увеличивать `serial`.
- Неправильные значения могут замедлить обновление зоны или вызвать проблемы с обслуживанием.

## Диагностика:
- `dig SOA example.com`
- `dig @ns1.example.com example.com SOA`.
