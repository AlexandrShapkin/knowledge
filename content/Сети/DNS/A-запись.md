A-[[Виды DNS-записей|запись]] связывает имя хоста с IPv4-адресом (32-битным). Это «классический» способ указать, куда направлять трафик по имени в сетях IPv4.

**Формат (типичная запись в zone-файле):**
```
www.example.com. 3600 IN A 93.184.216.34
```

**Как работает (резолв):**
- Рекурсивный резолвер запрашивает у авторитативного сервера A-запись для имени; получает IP и возвращает клиенту.
- TTL задаёт, как долго запись кешируется у рекурсивных резолверов.

## Типичные сценарии использования:
- Привязка веб-сервера, API, любого IPv4-хоста.
- Glue record: при делегировании, если NS находятся в делегируемой зоне (например, `ns1.example.com` как NS для `example.com`), в родительской зоне добавляют A-запись (glue), чтобы разрешить имя сервера при делегировании.

## Поведение и особенности:
- Можно иметь несколько A-записей для одного имени — это даёт простое DNS round-robin (простое распределение нагрузки / отказоустойчивость).
- DNS не обеспечивает health-checks: если один из IP упал, DNS всё ещё может выдавать этот IP, пока его не удалят/не изменят запись. Для реального failover используют балансировщики/health-checks + динамические записи на стороне провайдера.
- Для изменений IP учитывайте TTL: высокая TTL = долгое кэширование; для изменений временно понижают TTL.

## Практические рекомендации:
- Для статичных сервисов разумные TTL от 3600 до 86400; для частых переключений — временно снизить до 60–300 секунд перед изменением.
- При использовании собственного NS внутри зоны не забывайте про glue-записи у регистратора.
- Документируйте, какие A-записи соответствуют каким хостам (особенно если их много).

## Диагностика:
- `dig +short A www.example.com`
- `dig @ns1.example.com example.com A`
- `host www.example.com`
- `ping`/`traceroute` для проверки ответности IP (но ping использует сеть, а не DNS).
