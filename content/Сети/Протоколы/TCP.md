**TCP (Transmission Control Protocol)** — транспортный протокол с установлением соединения. Он обеспечивает надёжную передачу упорядоченного потока байтов между двумя процессами.

## Установление соединения

Соединение создаётся трёхэтапным рукопожатием:

1. клиент отправляет `SYN`;
2. сервер отвечает `SYN-ACK`;
3. клиент подтверждает `ACK`.

После этого стороны могут передавать данные в обоих направлениях.

## Надёжность

TCP использует:

- номера последовательности;
- подтверждения получения;
- повторную передачу потерянных сегментов;
- контроль порядка;
- контрольную сумму;
- управление потоком и перегрузкой.

Приложение получает непрерывный поток байтов. Границы отдельных записей TCP не сохраняет, поэтому прикладной протокол должен определять их самостоятельно.

## Порты и соединение

Соединение определяется четырьмя значениями:

```text
IP источника : порт источника → IP назначения : порт назначения
```

Сервер слушает известный порт, а клиент обычно использует временный исходящий порт.

## Завершение

Обычное завершение выполняется обменом `FIN` и `ACK`. Флаг `RST` немедленно прерывает соединение, например если порт закрыт или состояние соединения потеряно.

## Когда применять

TCP подходит для HTTP/1.1, HTTP/2, SSH, передачи файлов, баз данных и других сценариев, где важны полнота и порядок данных.

## Диагностика

Проверяют состояние сокетов, доступность порта, потери пакетов, повторные передачи и таймауты:

```bash
ss -tn
nc -vz example.com 443
tcpdump -nn tcp
```
