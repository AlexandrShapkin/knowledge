 `numpy.ndarray` — это основной тип данных в библиотеке NumPy, представляющий **n-мерный массив** (n-dimensional array).

По сути, это контейнер для однородных данных (все элементы одного типа), расположенных в памяти **непрерывным блоком**, что и даёт NumPy высокую производительность по сравнению с обычными списками Python.

## Ключевые свойства `ndarray`:

Он **типизированный**: все элементы имеют один `dtype` (например `int32`, `float64`). Это позволяет NumPy выполнять операции векторно и без лишних проверок.

Он **n-мерный**: может быть  
1D — вектор  
2D — матрица  
3D и выше — тензоры

Количество измерений хранится в [[Размерность массива (ndim)|ndim]], размеры — в [[Понятие формы массива (shape)|shape]].

Он хранит данные отдельно от метаданных. Сам объект `ndarray` содержит:  
– указатель на участок памяти с данными  
– информацию о форме (`shape`)  
– шагах (`strides`) — как перемещаться по памяти между элементами  
– типе данных (`dtype`)

За счёт `strides` один и тот же буфер памяти может быть представлен разными способами (срезы, транспонирование) **без копирования данных**.

Он поддерживает **векторизованные операции**: арифметика, сравнения и математика применяются сразу ко всему массиву, без явных циклов Python.

Важно понимать отличие от `list`:  
– `list` хранит ссылки на объекты Python  
– `ndarray` хранит «сырые» значения в плотном массиве  
– операции над `ndarray` выполняются в C-коде NumPy, а не в интерпретаторе

Минимальный пример:

```python
import numpy as np

a = np.array([1, 2, 3])
type(a)  # numpy.ndarray
```

Если упростить:  
`numpy.ndarray` — это низкоуровневый, эффективный и строго структурированный массив данных, на котором построена вся экосистема численных вычислений в Python (NumPy, SciPy, pandas, scikit-learn и др.).
