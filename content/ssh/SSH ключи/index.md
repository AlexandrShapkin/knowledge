## Назначение

SSH-ключи используются для аутентификации без передачи пароля серверу.
Пара ключей состоит из:

- приватного ключа (private key)
- публичного ключа (public key)

Приватный ключ хранится только на клиенте и никогда не должен передаваться другим лицам или копироваться на сервер.
Публичный ключ можно свободно распространять. Он размещается на удалённом сервере и используется для проверки подлинности клиента.

---
## Как работает аутентификация

Упрощённая схема:

```text
Клиент                                Сервер

id_ed25519
id_ed25519.pub  ------------------>  authorized_keys

SSH соединение --------------------> Проверка подписи
```

При подключении сервер проверяет, что клиент обладает соответствующим приватным ключом. Сам приватный ключ по сети не передаётся.

## Структура каталога ~/.ssh

Типичный набор файлов:

```text
~/.ssh/
├── config
├── known_hosts
├── authorized_keys
├── id_ed25519
└── id_ed25519.pub
```

### config

Клиентская конфигурация SSH.
Используется для создания псевдонимов хостов, выбора пользователя, ключа, порта и других параметров подключения.

Подробнее: [SSH Config](../SSH%20Config.md)

### known_hosts

Список известных серверов и их host key.
Используется для защиты от атак типа Man-In-The-Middle.

Подробнее: SSH Host Keys и known_hosts

### authorized_keys

Список публичных ключей, которым разрешён вход на сервер.

Подробнее: [Добавление публичного ключа на сервер](Добавление%20публичного%20ключа%20на%20сервер.md)

### id_ed25519

Приватный ключ.
Никогда не передавать другим лицам и не копировать на сервер.

### id_ed25519.pub

Публичный ключ.
Именно его содержимое добавляется в файл `authorized_keys`.

## Рекомендуемые алгоритмы ключей

### ED25519

Предпочтительный вариант для большинства случаев.

Преимущества:

- современный алгоритм;
- высокая производительность;
- короткие ключи;
- высокий уровень безопасности.

### RSA

Используется для совместимости со старыми системами.

Рекомендуемый размер ключа:

```text
RSA 4096
```

### ECDSA

Используется редко.

Обычно нет причин выбирать его вместо ED25519.

---

## Практические рекомендации

- Использовать ED25519 по умолчанию.
- Устанавливать парольную фразу на приватный ключ.
- Использовать SSH Agent.
- Хранить разные ключи для разных задач.
- Не использовать один и тот же ключ для всех серверов.
- Не передавать приватные ключи через мессенджеры, почту и облачные хранилища.
- Регулярно удалять неиспользуемые ключи из `authorized_keys`.
