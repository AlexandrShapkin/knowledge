## Назначение

ED25519 — современный алгоритм асимметричной криптографии, рекомендуемый для SSH-аутентификации в большинстве случаев.

Преимущества:

- высокий уровень безопасности;
- небольшие размеры ключей;
- быстрое создание и использование;
- поддерживается всеми современными версиями OpenSSH.

Для новых SSH-ключей рекомендуется использовать именно ED25519, если отсутствуют требования совместимости со старыми системами.

---

## Что будет создано

После генерации создаётся пара файлов:

```text
~/.ssh/id_ed25519
~/.ssh/id_ed25519.pub
```

### id_ed25519

Приватный ключ.
Используется для подтверждения личности клиента.
Никогда не должен передаваться другим лицам или копироваться на сервер.

### id_ed25519.pub

Публичный ключ.
Используется сервером для проверки подлинности клиента.
Может свободно распространяться.

## Базовая генерация ключа

Linux, macOS, WSL, Git Bash:

```bash
ssh-keygen -t ed25519
```

После запуска программа запросит:

```text
Enter file in which to save the key:
```

Путь сохранения ключа.

По умолчанию:

```text
~/.ssh/id_ed25519
```

Далее будет предложено задать парольную фразу:

```text
Enter passphrase:
```

После завершения будут созданы приватный и публичный ключи.

## Добавление комментария

Комментарий помогает определить назначение ключа.

Например:

```bash
ssh-keygen -t ed25519 -C "user@example.com"
```

Или:

```bash
ssh-keygen -t ed25519 -C "work-laptop"
```

Комментарий сохраняется внутри публичного ключа.

Пример:

```text
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAI... work-laptop
```

Комментарий не влияет на безопасность и используется только для идентификации.

---

## Использование парольной фразы

Рекомендуется защищать приватный ключ парольной фразой.

Пример:

```text
Enter passphrase:
```

При компрометации файла злоумышленник не сможет использовать ключ без знания парольной фразы.

Для удобства последующей работы обычно используется SSH Agent.

Подробнее: [[SSH Agent]]

---

## Увеличение стоимости подбора парольной фразы

[OpenSSH](https://www.openssh.com/) позволяет увеличить количество вычислений, необходимых для расшифровки приватного ключа.

Для этого используется параметр:

```bash
-a
```

Пример:

```bash
ssh-keygen -t ed25519 -a 100
```

Или:

```bash
ssh-keygen -t ed25519 -a 100 -C "work-laptop"
```

Чем больше значение, тем сложнее перебор парольной фразы.

Для современных систем значения 64–100 считаются разумным компромиссом между безопасностью и удобством.

## Указание собственного имени файла

По умолчанию создаётся:

```text
~/.ssh/id_ed25519
```

Для хранения нескольких ключей удобно использовать отдельные имена.

Пример:

```bash
ssh-keygen -t ed25519 -f ~/.ssh/github
```

Будут созданы файлы:

```text
~/.ssh/github
~/.ssh/github.pub
```

Другой пример:

```bash
ssh-keygen -t ed25519 -f ~/.ssh/prod-server
```

Рекомендуется использовать отдельные ключи для:

- GitHub;
- GitLab;
- рабочих серверов;
- личных серверов;
- критичных инфраструктур.

## Рекомендуемая команда

Для большинства случаев:

```bash
ssh-keygen -t ed25519 -a 100 -C "work-laptop"
```

Для отдельного ключа:

```bash
ssh-keygen -t ed25519 -a 100 -C "github" -f ~/.ssh/github
```

## Проверка содержимого публичного ключа

Просмотр ключа:

```bash
cat ~/.ssh/id_ed25519.pub
```

Пример результата:

```text
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIO... work-laptop
```

Именно эта строка добавляется в файл:

```text
~/.ssh/authorized_keys
```

на удалённом сервере.

Подробнее: [[Добавление публичного ключа на сервер]]

---

## Просмотр fingerprint

Fingerprint используется для идентификации ключа.

Показать fingerprint публичного ключа:

```bash
ssh-keygen -lf ~/.ssh/id_ed25519.pub
```

Пример:

```text
256 SHA256:xxxxxxxxxxxxxxxxxxxxxxxx work-laptop (ED25519)
```

Полезно при инвентаризации ключей и проверке соответствия ключей на различных системах.

---

## Проверка типа ключа

Просмотр информации о ключе:

```bash
ssh-keygen -lf ~/.ssh/id_ed25519.pub
```

Пример:

```text
256 SHA256:xxxxxxxxxxxxxxxxxxxxxxxx work-laptop (ED25519)
```

В конце строки указан используемый алгоритм.

---

## Изменение комментария существующего ключа

Изменить комментарий без пересоздания ключа:

```bash
ssh-keygen -c -f ~/.ssh/id_ed25519
```

Будет предложено указать новый комментарий.

---

## Изменение парольной фразы

Смена парольной фразы существующего ключа:

```bash
ssh-keygen -p -f ~/.ssh/id_ed25519
```

Будет запрошена текущая парольная фраза и новая.

---

## Проверка прав доступа

Для корректной работы OpenSSH рекомендуется:

```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519
chmod 644 ~/.ssh/id_ed25519.pub
```

Некорректные права могут привести к ошибкам аутентификации.

## Windows

Современные версии Windows содержат встроенный OpenSSH Client.

Проверка наличия SSH:

```powershell
Get-Command ssh
```

Генерация ключа:

```powershell
ssh-keygen -t ed25519 -a 100 -C "work-laptop"
```

Файлы будут созданы в каталоге:

```text
C:\Users\<USERNAME>\.ssh\
```

Собственное имя файла:

```powershell
ssh-keygen -t ed25519 -a 100 -f $env:USERPROFILE\.ssh\github
```

## Частые ошибки

### Перезаписан существующий ключ

Если указать существующее имя файла:

```bash
ssh-keygen -f ~/.ssh/id_ed25519
```

OpenSSH предложит подтверждение перезаписи.

Следует внимательно проверять путь перед подтверждением.

### Потерян приватный ключ

Восстановить приватный ключ невозможно.

Необходимо:

1. Сгенерировать новую пару ключей.
2. Заменить публичный ключ на всех серверах.

### Утеряна парольная фраза

Снять или восстановить парольную фразу невозможно.
Потребуется создание новой пары ключей.
