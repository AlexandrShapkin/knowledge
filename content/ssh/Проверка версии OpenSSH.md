## Назначение

Версия OpenSSH влияет на доступность алгоритмов ключей, параметров конфигурации и механизмов аутентификации. При диагностике несовместимости клиента и сервера сначала следует определить версии обеих сторон.

---

## Проверка версии клиента

```bash
ssh -V
```

Пример вывода:

```text
OpenSSH_9.9p1, OpenSSL 3.2.2
```

Команда пишет результат в стандартный поток ошибок, поэтому при перенаправлении вывода используется:

```bash
ssh -V 2>&1
```

---

## Проверка версии сервера

На сервере:

```bash
sshd -V
```

В некоторых сборках параметр `-V` отсутствует. Тогда можно проверить версию установленного пакета.

Debian и Ubuntu:

```bash
dpkg-query -W openssh-server
```

RHEL, AlmaLinux и Rocky Linux:

```bash
rpm -q openssh-server
```

Arch Linux:

```bash
pacman -Q openssh
```

---

## Получение баннера удалённого сервера

Без аутентификации:

```bash
nc host.example.com 22
```

Пример:

```text
SSH-2.0-OpenSSH_9.6p1 Ubuntu-3ubuntu13
```

Баннер может быть изменён или скрыт настройками сервера, поэтому он не всегда точно отражает установленную версию.

---

## Проверка поддерживаемых алгоритмов

Алгоритмы ключей:

```bash
ssh -Q key
```

Алгоритмы обмена ключами:

```bash
ssh -Q kex
```

Шифры:

```bash
ssh -Q cipher
```

MAC-алгоритмы:

```bash
ssh -Q mac
```

Итоговая конфигурация для конкретного хоста:

```bash
ssh -G host.example.com
```

---

## Практический порядок проверки

1. Проверить версию клиента через `ssh -V`.
2. Проверить версию серверного пакета.
3. Сравнить поддерживаемые алгоритмы через `ssh -Q`.
4. Посмотреть итоговые параметры через `ssh -G`.
5. Выполнить подключение с подробным выводом:

```bash
ssh -vvv user@host.example.com
```
