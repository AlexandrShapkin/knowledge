`CODE` — объём памяти, занятый исполняемым кодом процесса (text segment).

Кратко:

- Отражает размер сегмента кода в адресном пространстве процесса.
- Формируется на основе данных о сегменте `.text` и соответствующих отображениях памяти.
- Обычно является частью [[RES|RES]] при фактической загрузке в RAM.

Что включает:

- Исполняемый код основного бинарного файла.
- Код подключённых динамических библиотек.
- Read-only сегменты, связанные с выполнением программы.

Особенности:

- Как правило, доступен только для чтения.
- Часто разделяется между несколькими процессами.
- Обычно стабилен после запуска процесса.
- Влияет на [[VIRT|VIRT]], но вносит ограниченный вклад в рост памяти во время работы.

Используется для оценки объёма загруженного кода и анализа структуры адресного пространства процесса.
