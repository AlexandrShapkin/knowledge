`SHR` — объём разделяемой памяти в составе [[RES|RES]].

Кратко:

- Показывает часть resident-памяти, которая может быть разделена с другими процессами.
- Основан на данных о shared-страницах в адресном пространстве процесса.
- Является подмножеством `RES`.

Что включает:

- Страницы разделяемых библиотек.
- Отображённые файлы (mmap), доступные нескольким процессам.
- System V / POSIX shared memory.
- Другие общие readonly-сегменты.

Особенности:

- Эти страницы могут учитываться в RES нескольких процессов одновременно.
- Значение не означает, что память будет полностью освобождена при завершении процесса.
- Не равно объёму памяти, который «делится» в реальности — это оценка на уровне страниц.
- Точная доля реально разделяемых страниц определяется только через анализ `/proc/[pid]/smaps`.

Используется для приблизительной оценки доли общей памяти и анализа влияния библиотек и shared-сегментов на общий расход RAM.
