`CLS` — класс планирования процесса, определяющий алгоритм распределения CPU, применяемый ядром Linux.

Кратко:

- Показывает, каким планировщиком обслуживается процесс.
- Определяется при создании процесса или через `sched_setscheduler`.
- Влияет на правила вытеснения, квантование и приоритеты.

Основные классы:

- `SCHED_OTHER` (CFS) — стандартный класс. Использует Completely Fair Scheduler, распределяющий CPU пропорционально весу (зависит от `nice`).
- `SCHED_FIFO` — real-time без квантования. Процесс выполняется до блокировки, завершения или вытеснения задачей с более высоким RT-приоритетом.
- `SCHED_RR` — real-time с квантованием. Аналог FIFO, но с фиксированным тайм-слайсом.
- `SCHED_BATCH` — вариант CFS для фоновых вычислений с пониженным влиянием на интерактивные задачи.
- `SCHED_IDLE` — минимальный приоритет, выполняется только при отсутствии других runnable-задач.
- `SCHED_DEADLINE` — дедлайновый планировщик с параметрами runtime/period/deadline (не всегда отображается в `top`/`htop`).

Особенности:

- Real-time классы (`FIFO`, `RR`) всегда имеют приоритет над CFS.
- Для CFS учитывается `nice`, для real-time — `RTPRIO`.
- Неправильный выбор класса может вызвать starvation или деградацию интерактивности системы.

Отражает модель планирования, по которой процесс конкурирует за CPU, и используется для анализа CPU-bound нагрузок и настройки сервисов с требованиями к задержкам.
