Раздел CPU в `top`/`htop` отражает использование процессорного времени конкретным процессом. Эти поля описывают фактическое потребление CPU и его распределение между задачами. Данные формируются на основе счётчиков времени в `/proc/[pid]/stat` и информации планировщика.

Краткое пояснение к полям:

[%CPU](процент%20CPU.md) — доля процессорного времени, потреблённая процессом за интервал обновления.

- Рассчитывается как отношение использованного CPU-времени к прошедшему времени.
- Может превышать 100% в многопроцессорной системе (если используются несколько ядер).
- Показывает текущую активность процесса.

[TIME+](TIME+.md) — суммарное CPU-время процесса с момента запуска (в формате `часы:минуты:секунды.сотые`).

- Включает user time и system time.
- Отображается с повышенной точностью.
- Используется для оценки общего объёма вычислений.

[cumulative CPU time](cumulative%20CPU%20time.md) — накопленное процессорное время (user + system).

- Представляет общее количество времени, которое процесс фактически выполнялся на CPU.
- Не включает периоды ожидания.
- Является базой для расчёта `%CPU`.

[last used CPU (processor/core)](last%20used%20CPU.md) — номер последнего процессора (ядра), на котором выполнялся процесс.

- Отражает последнее назначение планировщиком.
- Полезно при анализе affinity и распределения нагрузки.
- Не гарантирует, что процесс продолжит выполняться на том же ядре.

Эти поля используются для анализа вычислительной нагрузки, выявления CPU-bound процессов и диагностики проблем с распределением процессорного времени.
