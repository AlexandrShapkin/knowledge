 Ниже перечислены основные параметры и метрики, которые обычно можно наблюдать через [[top|top]] и [[htop|htop]]. Конкретный набор зависит от версии, настроек и ядра Linux.

Системные агрегированные метрики (верхняя панель):

[[CPU|CPU]]:

- [[user (us)|user (us)]]
- [[system (sy)|system (sy)]]
- [[nice (ni)|nice (ni)]]
- idle (id)
- [[iowait (wa)|iowait (wa)]]
- hardware interrupt (hi)
- software interrupt (si)
- steal (st)
- guest (в некоторых версиях)
- загрузка по каждому ядру

Load Average:

- 1 минута
- 5 минут
- 15 минут

Задачи:

- общее количество процессов
- running
- sleeping
- stopped
- zombie

Память:

- total
- used
- free
- buffers
- cache
- available (в новых версиях)

Swap:

- total
- used
- free

Uptime системы.

Поля таблицы процессов (возможные колонки):

Идентификаторы:

- PID
- PPID
- TID (в режиме потоков)
- UID
- USER
- GROUP

Состояние:

- S (state: R, S, D, T, Z и др.)
- PRI (priority)
- NI (nice)
- RT priority (для realtime)
- scheduler policy

CPU:

- %CPU
- TIME+
- cumulative CPU time
- last used CPU (processor/core)

Память:

- VIRT (virtual memory size)
- RES (resident set size)
- SHR (shared memory)
- %MEM
- CODE
- DATA
- SWAP (в htop)

Планирование и приоритет:

- priority
- nice
- realtime priority
- scheduling class (CFS, FIFO, RR и др.)

I/O (зависит от поддержки ядра и версии):

- IO read rate
- IO write rate
- IO read bytes
- IO write bytes
- IO priority

Контекст:

- nTH (количество потоков)
- voluntary context switches
- non-voluntary context switches

Команда:

- COMMAND
- полный command line
- executable path

Дополнительно в htop:

- отображение процессов деревом
- отображение cgroup
- container (Docker/LXC, если доступно)
- namespace информация (в некоторых сборках)
- задержки планирования
- CPU affinity

Важно понимать, что `top` по умолчанию показывает ограниченный набор полей, но их можно включать/отключать через настройку колонок. `htop` обычно предоставляет более широкий и настраиваемый список. Полный перечень зависит от версии util-linux/procps-ng и сборки конкретного дистрибутива.
