[Load Average](Load%20Average.md) в Linux отражает количество процессов, которые **активно используют CPU** или **ожидают CPU / завершения I/O**.

Конкретно включаются процессы, находящиеся в [состояниях](../../../../Процессы/Состояния%20процессов.md):

- **R (running)** — процесс выполняется на CPU.
- **D (uninterruptible sleep)** — процесс ждёт завершения операции I/O (например, чтение/запись на диск).

Не учитываются процессы в состоянии `S` (sleeping) без ожидания ресурсов.

Таким образом, Load Average — это усреднённое число процессов в **очереди к CPU или в блокированном ожидании I/O**, рассчитанное с экспоненциальным скользящим средним для 1, 5 и 15 минут.

Практический смысл:

- Load Average = 1 на одноядерной системе → CPU полностью занят.
- Load Average < количество ядер → система в норме, есть резерв CPU.
- Load Average > количество ядер → процессы ждут CPU/I/O, возможны задержки.

Это агрегированный показатель, который отражает **общую загруженность системы**, а не только CPU.
