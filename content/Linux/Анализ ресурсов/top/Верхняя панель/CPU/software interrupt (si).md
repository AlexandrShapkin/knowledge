`software interrupt` (`si`) — процент времени CPU, затраченный на обработку программных прерываний (softirq).

Программные прерывания инициируются ядром для обработки задач, которые не требуют немедленного внимания, но должны выполняться быстро. Примеры: обработка сетевых пакетов, таймеров, планировщика задач, deferred work.

Практическая интерпретация:

- Низкий `si` — нормальная работа системы.
- Повышенный `si` — активная работа с сетевыми пакетами, планировщиком, асинхронными задачами ядра.
- Чрезмерно высокий `si` может приводить к задержкам пользовательских процессов и падению производительности.

Анализируется совместно с [hi](hardware%20interrupt%20%28hi%29.md) и общей загрузкой CPU для диагностики системной активности и источников нагрузки.
