Графа идентификаторов в списке процессов [[top|top]]/[[htop|htop]] отражает структурные и контекстные параметры процесса. Эти поля не связаны с нагрузкой или ресурсами напрямую, а описывают положение процесса в системе: его идентичность, иерархию и субъект безопасности.

Данные формируются на основе информации из `/proc` и внутренних структур ядра (task_struct).

Краткое пояснение к полям:

- **[[PID|PID]]** — уникальный идентификатор процесса в текущем PID namespace. Используется ядром для адресации процесса (сигналы, управление, /proc/).
- **[[PPID|PPID]]** — идентификатор родительского процесса. Показывает, кем был создан текущий процесс. Формирует дерево процессов.
- **[[TID|TID]]** — идентификатор потока выполнения. В Linux потоки реализованы как задачи (tasks). При отображении потоков каждый имеет собственный TID; у главного потока он совпадает с PID.
- **[[UID|UID]]** — числовой идентификатор пользователя, от имени которого выполняется процесс. Определяет права доступа в рамках модели DAC.
- **[[USER|USER]]** — строковое представление UID (имя пользователя из системы учётных записей). Человекочитаемая форма UID.
- **[[GROUP|GROUP (GID)]]** — числовой идентификатор основной группы процесса. Используется совместно с UID для проверки прав доступа к ресурсам. Дополнительные группы обычно не отображаются в этих колонках.

С практической точки зрения эти поля используются для анализа дерева процессов, проверки контекста безопасности и диагностики проблем с правами доступа.
