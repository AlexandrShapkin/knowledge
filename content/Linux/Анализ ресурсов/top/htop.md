`htop` — интерактивный консольный монитор процессов для Linux/Unix-систем. Используется для оперативного анализа загрузки системы и управления процессами в реальном времени. Является более функциональной альтернативой [[top|top]].

### Назначение

Основные задачи:

- просмотр текущей загрузки CPU и памяти
- анализ списка процессов
- поиск и фильтрация процессов
- отправка сигналов процессам (kill, terminate и др.)
- изменение приоритета (nice)

Применяется при диагностике высокой нагрузки, утечек памяти, зависших процессов.

### Запуск

```
htop
```    

### Интерфейс

Интерфейс разделён на две части:

1. Верхняя панель — агрегированные метрики:
    - CPU по ядрам
    - RAM
    - Swap
    - Uptime
    - Load Average
2. Нижняя часть — таблица процессов.

Навигация осуществляется стрелками, функциональные клавиши отображаются внизу экрана.

### Основные клавиши

- `F3` — поиск
- `F4` — фильтр
- `F5` — дерево процессов
- `F6` — сортировка
- `F9` — отправка сигнала
- `F10` — выход

### Особенности

- Поддержка цветового отображения.
- Работа мышью.
- Гибкая настройка колонок и метрик.
- Возможность отображения процессов в виде дерева.

### Отличия от top

По сравнению с `top`:

- более удобная навигация
- интерактивное управление без ввода команд вручную
- наглядное отображение ресурсов
- поддержка горизонтальной и вертикальной прокрутки

### Примечание

`htop` удобен для оперативного анализа, но для глубокой диагностики производительности обычно используются дополнительные инструменты (perf, atop, iotop и др.).
