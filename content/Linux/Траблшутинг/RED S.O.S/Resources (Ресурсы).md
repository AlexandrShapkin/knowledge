Цель: определить узкие места аппаратных или основных ресурсов.

**CPU / Load Average**

- LoadAverage — не процент, а длина очереди процессов.
- Сравнивать _LA_ с количеством ядер (`nproc`).
- Высокий _LA_ при низком %CPU user — признак I/O-голодания.  

Команды:
- [htop](../../Анализ%20ресурсов/top/htop.md):
```bash
htop cat /proc/loadavg
```

**Память**

- Смотреть _available_, а не _free_.
- Swap-активность (`si`/`so`) указывает на исчерпание RAM.  

Команда:
```bash
free -h
```

**Диски**

- Проверить свободное место (`df -hT | grep -v tmpfs`).
- Проверить производительность (`iostat -dx 1 3`).
- `%util` > 80% и высокий `await` → узкое место дисковой подсистемы.  

Команды:
```bash
df -hT | grep -v tmpfs iostat -dx 1 3
```
