PTR-[запись](Виды%20DNS-записей.md) используется в зонах обратного DNS (reverse DNS). Она связывает IP-адрес с доменным именем (обратное преобразование).

**Формат (zone-файл):**
```
34.216.184.93.in-addr.arpa. 86400 IN PTR www.example.com.
```

**Как работает:**
- Для IPv4 используется зона `in-addr.arpa`, где байты IP-адреса указываются в обратном порядке.
- Для IPv6 используется зона `ip6.arpa`, где адрес разбивается по нибблам (каждый hex-символ).
- Запрос PTR возвращает имя, ассоциированное с IP.

## Сценарии использования:
- Почтовые сервера (SMTP часто проверяет PTR отправителя).
- Отладка и администрирование сетей.
- Проверка сетевых подключений в некоторых протоколах.

## Особенности и ограничения:
- PTR должны соответствовать A/AAAA-записям (рекомендуется симметрия).
- PTR настраивается владельцем IP-адреса (обычно провайдер).
- Отсутствие корректного PTR может приводить к проблемам с почтой (спам-фильтры).

## Диагностика:
- `dig -x 93.184.216.34`
- `host 93.184.216.34`.
