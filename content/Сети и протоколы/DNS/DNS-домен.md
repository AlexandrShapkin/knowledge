**DNS-домен** — это область в системе доменных имён (Domain Name System), которая используется для идентификации ресурсов в сети через человеко-читаемые имена.
## Основные элементы DNS-домена
1. **Иерархия**
    - Система DNS построена по деревообразной структуре.
    - На самом верху находится **корневой домен** (`.`).
    - Ниже расположены **домены верхнего уровня** (TLD) — например: `.com`, `.org`, `.ru`.
    - Ещё ниже идут **вторичные домены** (например, `example.com`).
    - Внутри них могут создаваться **поддомены** (`mail.example.com`, `shop.example.com`).
2. **Доменное имя**
    - Представляет собой строку, состоящую из меток, разделённых точками.
    - Чтение идёт справа налево: от TLD к поддоменам.
    - Пример: в имени `www.example.com` часть `com` — это TLD, `example` — второй уровень, `www` — поддомен.
3. **Зоны DNS**
    - Домен может быть разделён на **зоны** — части пространства имён, за которые отвечает определённый DNS-сервер.
    - Например, за `example.com` отвечает один DNS-сервер, а за `sub.example.com` — другой.

## Назначение DNS-доменов
- Преобразование человеко-читаемых имён в IP-адреса.
- Упрощение доступа к ресурсам: легче запомнить `google.com`, чем `142.250.186.206`.
- Организация структуры интернета и делегирование ответственности за разные зоны.
- Возможность создания независимых пространств имён внутри компаний или сервисов.

## Пример
Имя `mail.example.com`:
- `.com` — домен верхнего уровня.
- `example` — домен второго уровня, зарегистрированный владельцем.
- `mail` — поддомен, который может указывать на почтовый сервер.

**Итог:**  
DNS-домен — это элемент иерархической системы, который служит для адресации и упрощённой навигации в сети. Он позволяет разделять пространство имён, делегировать управление и связывать имена с IP-адресами через DNS-записи.
