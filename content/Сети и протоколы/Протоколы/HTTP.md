**HTTP (Hypertext Transfer Protocol)** — прикладной протокол обмена запросами и ответами между клиентом и сервером. Он используется браузерами, API и reverse proxy.

## Запрос и ответ

Клиент отправляет метод, адрес ресурса, заголовки и при необходимости тело. Сервер возвращает код состояния, заголовки и тело ответа.

```text
GET /health HTTP/1.1
Host: example.com

HTTP/1.1 200 OK
Content-Type: text/plain

ok
```

## Методы

- `GET` — получить ресурс;
- `POST` — передать данные или создать ресурс;
- `PUT` — заменить ресурс;
- `PATCH` — частично изменить ресурс;
- `DELETE` — удалить ресурс;
- `HEAD` — получить только заголовки.

## Коды состояния

- `2xx` — успешная обработка;
- `3xx` — перенаправление;
- `4xx` — ошибка запроса клиента;
- `5xx` — ошибка сервера.

## Транспорт

HTTP/1.1 и HTTP/2 обычно работают поверх TCP. HTTP/3 использует QUIC поверх UDP. HTTPS — это HTTP, защищённый TLS, обычно на порту 443.

HTTP не хранит состояние между запросами. Сеансы реализуются cookie, токенами или логикой приложения.

## Диагностика

```bash
curl -v https://example.com/
curl -I https://example.com/
```

Проверяют DNS, TLS, код состояния, заголовки, перенаправления и таймауты.
