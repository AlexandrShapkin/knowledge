**DNS (Domain Name System)** преобразует доменные имена в данные, необходимые приложениям, прежде всего в IPv4- и IPv6-адреса. Благодаря DNS пользователю не нужно запоминать адрес каждого сервера.

## Участники запроса

- **Клиентский resolver** формирует запрос от имени приложения.
- **Рекурсивный resolver** ищет ответ и кэширует его.
- **Авторитетный сервер** хранит записи конкретной DNS-зоны.
- **Корневые и TLD-серверы** направляют resolver к авторитетному серверу.

## Основные типы записей

| Тип | Назначение |
|---|---|
| A | IPv4-адрес |
| AAAA | IPv6-адрес |
| CNAME | Псевдоним другого имени |
| MX | Почтовый сервер домена |
| NS | Авторитетный DNS-сервер |
| TXT | Текстовые данные и подтверждения |
| PTR | Обратное преобразование адреса в имя |

## Транспорт и кэширование

Обычные запросы используют UDP-порт 53. TCP применяется для больших ответов, передачи зон и повторного запроса после усечённого UDP-ответа.

Каждая запись имеет TTL. Пока TTL не истёк, resolver может вернуть кэшированный ответ без обращения к авторитетным серверам. Поэтому изменение записи распространяется не мгновенно.

## Диагностика

```bash
dig example.com A
dig @1.1.1.1 example.com
dig +trace example.com
```

Проверяют доступность resolver, корректность записей, делегирование зоны, TTL и различия ответов разных серверов.
