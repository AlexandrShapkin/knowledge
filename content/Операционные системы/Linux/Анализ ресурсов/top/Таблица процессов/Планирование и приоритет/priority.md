`PR` — числовой приоритет процесса, используемый планировщиком CPU для определения порядка выполнения.

Кратко:

- Показывает текущий приоритет задачи с точки зрения ядра.
- Формируется на основе `nice` (для обычных задач) или статического RT-приоритета (для real-time).
- Меньшее числовое значение означает более высокий приоритет.

Что включает:

- Для обычных задач класса CFS — базируется на значении `nice` и внутреннем весе планировщика.
- Для real-time задач (`SCHED_FIFO`, `SCHED_RR`) — соответствует их RT-приоритету.
- Может учитывать внутренние корректировки ядра (динамика виртуального времени в CFS).

Особенности:

- Для CFS обычно отображается как `20 + nice` (при `nice=0` приоритет около 20).
- Для real-time задач значения находятся ниже обычных (в `top` часто <20).
- Само по себе не гарантирует фиксированную долю CPU — поведение зависит от класса планирования.
- Может изменяться при `renice` или смене scheduling class.

Отражает относительное положение процесса в очереди выполнения CPU. Используется для анализа CPU-bound процессов, настройки приоритетов сервисов и диагностики starvation.
