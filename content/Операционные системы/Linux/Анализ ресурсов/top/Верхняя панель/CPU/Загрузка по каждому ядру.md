Это визуальное представление того, как используется каждый логический CPU (ядро). Каждое поле/полоска показывает распределение времени ядра по категориям: [user](user%20%28us%29.md), [system](system%20%28sy%29.md), [nice](nice%20%28ni%29.md), [iowait](iowait%20%28wa%29.md), [irq](hardware%20interrupt%20%28hi%29.md)/[softirq](software%20interrupt%20%28si%29.md), [steal](steal%20%28st%29.md), [guest](guest.md), [idle](idle%20%28id%29.md).

Ключевые моменты:

- Полоска показывает соотношение активного (`user`, `system`) и простого (`idle`) времени для конкретного ядра.
- Цветовое кодирование ([htop](../../htop.md)) помогает быстро оценить тип нагрузки: пользовательские процессы, системные процессы, I/O ожидания и т.д.
- Позволяет выявить перекосы нагрузки: некоторые ядра могут быть полностью заняты, а другие простаивать.
- Полезно для диагностики узких мест при многопоточных приложениях и балансировке процессов планировщиком ядра.

То есть это не отдельная метрика в процентах, а агрегированная визуализация всех CPU-параметров по отдельным ядрам.
