Это визуальное представление того, как используется каждый логический CPU (ядро). Каждое поле/полоска показывает распределение времени ядра по категориям: [[user (us)|user]], [[system (sy)|system]], [[nice (ni)|nice]], [[iowait (wa)|iowait]], [[hardware interrupt (hi)|irq]]/[[software interrupt (si)|softirq]], [[steal (st)|steal]], [[guest]], [[idle (id)|idle]].

Ключевые моменты:

- Полоска показывает соотношение активного (`user`, `system`) и простого (`idle`) времени для конкретного ядра.
- Цветовое кодирование ([[../../htop]]) помогает быстро оценить тип нагрузки: пользовательские процессы, системные процессы, I/O ожидания и т.д.
- Позволяет выявить перекосы нагрузки: некоторые ядра могут быть полностью заняты, а другие простаивать.
- Полезно для диагностики узких мест при многопоточных приложениях и балансировке процессов планировщиком ядра.

То есть это не отдельная метрика в процентах, а агрегированная визуализация всех CPU-параметров по отдельным ядрам.
