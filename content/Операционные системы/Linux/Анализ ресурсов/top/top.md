`top` — стандартная консольная утилита для мониторинга процессов в Unix/Linux-системах. Входит в базовый набор практически любого дистрибутива и используется для анализа текущего состояния системы в реальном времени.

### Назначение

Основные задачи:

- просмотр загрузки CPU
- анализ использования памяти
- наблюдение за активными процессами
- сортировка процессов по различным метрикам
- отправка сигналов процессам
- изменение приоритета (nice)

Чаще всего применяется для быстрой диагностики высокой нагрузки или зависших процессов.

### Запуск

```
top
```

Дополнительно:
```
top -u username
top -p PID
```

### Интерфейс

Экран состоит из двух частей:

1. Сводка состояния системы:
    - Uptime
    - Load Average
    - количество задач
    - распределение CPU
    - использование памяти и swap
2. Таблица процессов.

Обновление происходит периодически (по умолчанию каждые 3 секунды).

### Основные команды (во время работы)

- `h` — справка
- `k` — отправить сигнал процессу
- `r` — изменить nice
- `P` — сортировка по CPU
- `M` — сортировка по памяти
- `T` — сортировка по времени
- `1` — показать загрузку по ядрам
- `q` — выход

### Особенности

- Минимальные зависимости, доступен почти всегда.
- Подходит для работы по SSH без дополнительных утилит.
- Менее интерактивен по сравнению с [htop](htop.md).
- Требует ручного ввода команд без визуальных подсказок.

### Примечание

`top` — базовый инструмент первичной диагностики. Для более удобного интерактивного анализа часто используется `htop`, однако `top` остаётся стандартом и гарантированно присутствует в системе.
