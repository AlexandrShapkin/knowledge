## Просмотр inode
Посмотреть номер [[inode]] можно так:
```bash
ls -i file.txt
```

Вывод покажет номер inode перед именем файла:
```
123456 file.txt
```

Чтобы увидеть inode всех файлов в каталоге:
```bash
ls -lai
```

Опции:
- `-l` — длинный формат (права, владелец, время)
- `-a` — включая скрытые файлы
- `-i` — показать номер inode
## Поиск файлов по inode
Если известен inode, можно найти связанное имя файла:
```bash
find . -inum 123456
```

Если нужно удалить файл с этим inode:
```bash
find . -inum 123456 -delete
```
или
```bash
find . -inum 123456 -exec rm -i {} \;
```
(второй вариант — с подтверждением)

Это особенно полезно, когда файл имеет «странное» имя, содержащее непечатаемые символы или пробелы в конце.
## Удаление файлов по inode
1. Сначала узнаём inode:
```bash
ls -i
```
2. Затем удаляем через `find`, как показано выше.  
    Это обходит проблему, если имя файла невозможно ввести в терминале.
## Проверка использования inode
Проверить общее количество inode и сколько из них занято:
```bash
df -i
```

Вывод покажет, сколько inode доступно на каждой файловой системе.  
Если inode закончились — новые файлы создать невозможно, даже если свободное место на диске есть.
## Информация о inode файла
Чтобы подробно посмотреть метаданные inode:
```bash
stat file.txt
```

Пример вывода:
```
  File: file.txt
  Size: 2048       Blocks: 8          IO Block: 4096 regular file
Device: 802h/2050d Inode: 123456      Links: 2
Access: (0644/-rw-r--r--)  Uid: (1000/user)   Gid: (1000/user)
Access: 2025-10-04 12:34:56
Modify: 2025-10-03 09:00:00
Change: 2025-10-03 09:00:00
 Birth: -
```
## Практические советы
- Если при удалении файла появляется ошибка вроде `device or resource busy`, значит файл всё ещё используется процессом. Узнать его можно через:
```bash
lsof | grep deleted
```
или:
```bash
lsof /path/to/mountpoint
```

- Чтобы освободить место, нужно завершить процесс, который держит удалённый файл открытым.
- inode потребляются не только файлами, но и каталогами, устройствами, сокетами и FIFO.
## Сводка по основным командам

|Цель|Команда|Примечание|
|---|---|---|
|Показать inode|`ls -i`|Номер inode перед именем|
|Подробная инфо|`stat file`|Вся метаинформация|
|Поиск по inode|`find . -inum N`|Можно ограничить каталог|
|Удалить по inode|`find . -inum N -delete`|Удобно для «битых» имён|
|Проверить использование inode|`df -i`|Показывает занятость inode|
|Проверить процессы, держащие файл|`lsof`|Помогает при ошибках удаления|
