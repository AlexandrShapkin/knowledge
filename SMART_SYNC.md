# Smart Sync

Команда:

```bash
npm run sync
```

Она выполняет следующий цикл:

1. Коммитит текущие локальные изменения.
2. Создаёт локальную резервную ветку `backup/smart-sync-*`.
3. Получает актуальное состояние текущей ветки из `origin`.
4. Выполняет обычный трёхсторонний Git merge.
5. Автоматически объединяет конфликтующие Markdown-файлы.
6. Отправляет результат в удалённую ветку без force push.
7. При гонке с новым удалённым коммитом повторяет fetch, merge и push до трёх раз.

По умолчанию синхронизируется текущая ветка. Для явного выбора:

```bash
npm run sync -- --branch v4
```

Другой remote:

```bash
npm run sync -- --remote upstream --branch v4
```

Собственное сообщение локального коммита:

```bash
npm run sync -- --message "Update SSH notes"
```

## Разрешение Markdown-конфликтов

Алгоритм использует базовую, локальную и удалённую версии файла.

- Изменения в разных участках объединяются стандартным трёхсторонним merge.
- Списки `tags`, `aliases` и `cssclasses` во frontmatter объединяются без дублей.
- Независимые элементы Markdown-списков объединяются без дублей.
- Изменения разных предложений одного абзаца объединяются на уровне предложений.
- При неоднозначном изменении одного блока сохраняются обе версии: сначала локальная, затем удалённая.
- Для конфликтов не-Markdown-файлов выбирается локальная версия.

Сведения о неоднозначных решениях сохраняются вне рабочего дерева:

```text
.git/smart-sync/<timestamp>/
```

Перед изменением истории создаётся резервная ветка. Для возврата:

```bash
git reset --hard backup/smart-sync-<timestamp>
```

## Ограничения

Команда останавливается, если уже выполняется merge, rebase, cherry-pick или revert. Сначала текущую Git-операцию нужно завершить либо отменить.

Автоматическое объединение не понимает смысл текста. Оно предотвращает тихую потерю обеих версий, но после сложного конфликта заметку следует просмотреть.
