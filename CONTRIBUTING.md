# Инструкция для контрибьюторов

Спасибо за интерес к базе знаний! Ниже описан безопасный способ внести изменения без необходимости быть Contributor.

---

## 1. Форк репозитория

1. Зайдите на страницу репозитория на GitHub.
2. Нажмите **Fork** в правом верхнем углу.
3. Получите полную копию репозитория в своём аккаунте.

## 2. Клонирование форка

```bash
git clone https://github.com/<your-username>/<repo>.git
cd <repo>
git remote add upstream https://github.com/AlexandrShapkin/knowledge.git
```

* `upstream` — это оригинальный репозиторий, чтобы синхронизироваться с последними изменениями.

## 3. Создание отдельной ветки для изменений

```bash
git checkout -b my-changes
```

* Работайте только в этой ветке. Так проще будет создать Pull Request и избежать конфликтов.

## 4. Редактирование заметок

1. Откройте папку репозитория в Obsidian.
2. Добавляйте или редактируйте заметки.
3. Сохраняйте изменения.

## 5. Синхронизация с оригинальной базой

Перед пушем убедитесь, что у вас актуальная версия:

```bash
git fetch upstream
git checkout v4
git merge upstream/v4
git checkout my-changes
git rebase v4
```

* Это позволит избежать конфликтов при создании Pull Request.

## 6. Пуш изменений в форк

```bash
git push origin my-changes
```

## 7. Создание Pull Request

1. Перейдите на GitHub в свой форк.
2. Нажмите **Compare & pull request**.
3. Опишите изменения.
4. Отправьте PR в оригинальный репозиторий в ветку `v4`.

* Владелец репозитория проверяет PR и сливает изменения, если всё в порядке.

## 8. Рекомендации

* Работайте в отдельной ветке для каждого изменения.
* Перед PR подтягивайте последние изменения из `upstream/v4`.
* В описании PR указывайте, что изменено или добавлено, чтобы обзор был проще.
